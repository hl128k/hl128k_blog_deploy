在使用Serv00服务器时，虽然通过大佬的教程部署了许多应用并使用pm2来保持应用活跃，但由于重启和清除定时任务，进程仍会时不时地崩溃。@saika提供了一种进程保活的最终解决方案，解决了这个问题。按照@saika的示例，部署fuclaude的步骤包括：在面板Port reservation中添加端口，使用SSH在新建域名目录的public_nodejs路径下创建并赋予执行权限的update_to_github.sh脚本，执行脚本下载最新的fuclaude，接着将app.js和package.json放入public_nodejs目录，执行npm install，并通过网页监控应用以保持其活跃。其他应用可以类似地替换命令进行保活。

Ubuntu中docker防火墙处理

发表于 2024-08-14 更新于 2024-08-27 分类于 技术教程 ， 部署
在使用VPS和Docker部署项目时，由于VPS没有外部防火墙，决定使用Ubuntu的ufw防火墙来管理端口。虽然docker启动和nginx映射看似正常，但发现docker忽略了ufw的设置，直接修改了iptables，使得端口依然可以访问。尝试禁用docker对iptables的修改权限导致docker内部通信失败，nginx无法访问。最终解决方案是使用默认的ufw设置，禁止所有端口开放，并通过统一网络将docker容器连接，端口不对外开放，统一使用nginx容器处理，并通过服务名称访问。

Docker 部署 Gitea Actions

发表于 2024-07-19 更新于 2024-08-27 分类于 技术教程 ， Gitea
在尝试使用Gitea的Actions进行本地部署时，作者总结了部署过程中的一些关键点和注意事项。Gitea的Actions能够兼容GitHub Actions，部署时主要涉及Docker-Compose配置。配置过程中，需要先生成`act_runner`的配置文件，通过在容器内执行`act_runner generate-config`命令来完成，同时要注意网络配置的问题，特别是`GITEA_INSTANCE_URL`的设置。由于`act_runner`生成的新容器可能无法直接连接到`docker-compose`中的Gitea实例，因此需要使用宿主地址或配置外部网络。建议在Docker网络中创建固定IP以确保网络连接正常。
